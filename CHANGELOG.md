# 更新日志

## 1.2.9

- 按 AstrBot 官方文档改为高优先级全消息事件监听，并在插件内解析 `/卡拉彼丘` 与 `/klbq` 前缀
- 移除命令参数绑定，避免事件在进入处理函数前被路由层丢弃
- 命中后停止事件传播，避免查询命令继续进入其他聊天插件

## 1.2.8

- 将备用命令从命令组 `/klbq 查 关键词` 改为普通命令 `/klbq 关键词`
- 移除命令组依赖，避免部分环境命令组注册后仍不触发

## 1.2.7

- 修复备用命令组定义位置错误导致的 `'function' object has no attribute 'command'` 加载失败

## 1.2.6

- 回退中文命令处理函数为 AstrBot 更稳定支持的单参数签名
- 新增备用命令组 `/klbq 查 关键词`，用于规避部分环境中文命令未触发的问题
- 查询处理拆为公共入口，中文命令和备用命令共用同一逻辑

## 1.2.5

- 移除 AstrBotConfig 直接导入，提升不同 AstrBot 版本的插件加载兼容性
- 命令参数改为可变参数接收，避免参数解析差异导致命令无响应

## 1.2.4

- 增加查询链路后台日志，覆盖命令接收、别名解析、页面命中、HTML 获取、字段解析、图片渲染和文本回退
- 增加主命令异常兜底，查询失败时会回复错误提示并将堆栈写入后台日志
- 扩展页面候选标题查询，提升带间隔符/空格标题的命中率

## 1.2.3

- 收紧图片渲染视口与卡片宽度，改为居中自适应布局
- 进一步减少右侧空白与内容裁切风险

## 1.2.2

- 过滤“卡拉彼丘画师协会”等非角色稳定字段，减少多余栏目
- 去掉默认角色输出中的“画师”字段，避免误展示非稳定信息

## 1.2.1

- 修复图片渲染右侧出现大块空白、内容被裁切的问题
- 重构卡片 HTML 模板，增加完整 HTML 包裹、固定视口宽度和更安全的网格换行

## 1.2.0

- 扩展内置角色别名表，覆盖更多角色常用称呼
- 支持「角色武器 / 角色的武器」动态解析到对应武器页面
- 增强武器数值提取，新增分距离部位伤害、基础伤害、部位系数、拉栓、换弹、蓄力等手感数据
- 放宽单字段展示长度，避免武器伤害矩阵被截断

## 1.1.1

- 修复角色页面武器字段误抓阵营/导航链接的问题
- 所有角色在识别到武器时都会提示对应武器查询命令
- 调整图片渲染参数与卡片背景，减少异常空白

## 1.1.0

- 将查询结果改为图片卡片输出，整体风格参考 Yunzai 项目常见卡片样式
- 新增可配置的详情链接发送开关
- 增加角色/武器结构化字段输出，包含伤害、射速、移速、弹匣容量等信息
- 新增别名支持，内置常用别名映射，并支持在配置中自定义扩展
- 为心夏查询补充武器提示：可继续查询「空境」
- 补充 README 与插件配置项

## 1.0.0

- 初始版本
- 支持通过 `/卡拉彼丘 关键词` 查询卡拉彼丘 Wiki 条目
- 支持获取页面简介与详情链接
